Use this skill for React work.

Guidelines:
- Inspect existing component patterns before editing.
- Prefer composition and colocated state over broad abstractions.
- Keep exported prop types explicit when they are part of a public component API.
- Use semantic HTML first, ARIA only when semantics need support.
- Check loading, empty, error, disabled, keyboard, and responsive states.
- Avoid unnecessary memoization; optimize after identifying a real render problem.
- For tests, exercise user-visible behavior rather than implementation details.
